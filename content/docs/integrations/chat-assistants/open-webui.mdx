## Configuration Steps

### Option 1: Docker Deployment

```bash
docker run -d \
  -p 3000:8080 \
  -v open-webui:/app/backend/data \
  -e OPENAI_API_BASE_URL="http://your-nexusgate-server:8080/api/v1" \
  -e OPENAI_API_KEY="your-nexusgate-api-key" \
  --name open-webui \
  ghcr.io/open-webui/open-webui:main
```

Replace the configuration values with <ServerUrl path="/api/v1" /> and <ApiKeyLink />.

### Option 2: Docker Compose

Create `docker-compose.yml`:

```yaml
version: '3'
services:
  open-webui:
    image: ghcr.io/open-webui/open-webui:main
    ports:
      - "3000:8080"
    volumes:
      - open-webui:/app/backend/data
    environment:
      - OPENAI_API_BASE_URL=http://your-nexusgate-server:8080/api/v1
      - OPENAI_API_KEY=your-nexusgate-api-key
      - WEBUI_AUTH=true
    restart: unless-stopped

volumes:
  open-webui:
```

Start the service:

```bash
docker compose up -d
```

## Advanced Configuration

### Environment Variables

| Variable | Description | Example |
|----------|-------------|---------|
| `OPENAI_API_BASE_URL` | API address | `http://server:8080/api/v1` |
| `OPENAI_API_KEY` | API key | `sk-xxx` |
| `WEBUI_AUTH` | Enable authentication | `true` |
| `WEBUI_SECRET_KEY` | JWT secret | `random-string` |
| `DEFAULT_MODELS` | Default model | `gpt-4o` |
| `MODEL_FILTER_ENABLED` | Enable model filtering | `true` |
| `MODEL_FILTER_LIST` | Allowed models | `gpt-4o;gpt-4o-mini` |

### Configure RAG

Configure RAG in admin settings:

```yaml
# Document settings
chunk_size: 500
chunk_overlap: 50
embedding_model: text-embedding-3-small

# Retrieval settings
top_k: 5
relevance_threshold: 0.7
```

### Configure Webhooks

Support for webhook notifications:

```yaml
webhooks:
  - url: https://your-webhook-url
    events:
      - message.created
      - chat.created
    secret: webhook-secret
```

## FAQ

### Q: Cannot connect to API?

1. Check `OPENAI_API_BASE_URL` format
2. Ensure NexusGate is accessible from Docker container
3. Use `host.docker.internal` to access host services

```bash
# If NexusGate is on host
-e OPENAI_API_BASE_URL="http://host.docker.internal:8080/api/v1"
```

### Q: Model list is empty?

1. Check API connection
2. Manually configure model list
3. Enable model filtering and set allowed list

### Q: Poor RAG retrieval results?

1. Adjust chunk_size
2. Optimize document format
3. Choose a better embedding model

### Q: How to backup data?

```bash
# Backup Docker volume
docker run --rm \
  -v open-webui:/data \
  -v $(pwd):/backup \
  alpine tar czf /backup/open-webui-backup.tar.gz /data
```

### Q: How to update version?

```bash
docker pull ghcr.io/open-webui/open-webui:main
docker compose up -d
```
